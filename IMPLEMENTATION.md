# BiasharaPOS — Implementation Status

> **Live at https://biashara-pos.com** · Status as of 2026-05-30
>
> The shipping site is **Astro 4** with a static-only output (`output: 'static'`).
> Sections live as individual `.astro` components under `src/components/` and are
> composed by `src/pages/index.astro`. Per-vertical landing pages are generated by
> the dynamic route `src/pages/pos-for-[vertical].astro` (5 paths via
> `getStaticPaths`). Interactive bits (industry switcher, FAQ accordion, nav scroll
> shadow, mobile hamburger, reveal-on-scroll, bilingual EN/SW toggle, GA4 conversion
> tracking) are inline `<script is:inline>` blocks inside `src/layouts/Layout.astro`.
> The chat widget streams from the live Railway agent and lives in
> `src/components/ChatWidget.astro`.
>
> The original designer handoff bundle (HTML / CSS / vanilla JS prototype) lives in
> `Biashara-POS Website/design_handoff_biashara_landing/` for reference only and is
> not shipped.

## Stack

| | |
|---|---|
| Framework | Astro 4 (static, zero JS by default) |
| Styling | Plain CSS — design tokens as CSS custom properties in `src/styles/landing.css` |
| Images | `astro:assets` `<Image />` → responsive WebP with srcset/sizes |
| Fonts | Google Fonts `<link>` — Plus Jakarta Sans (400/500/600/700/800 + italic 500/600) |
| SEO | `@astrojs/sitemap` (3.2.1), JSON-LD in `Layout.astro`, canonical per page |
| Analytics | GA4 (`G-74MCNRL92X`) with custom conversion events |
| Chat backend | Railway-hosted WhatsApp agent (`whatsapp-agent/`) — SSE streaming via `PUBLIC_CHAT_API_URL/chat/stream` |
| Deploy | GitHub Actions → GitHub Pages → `biashara-pos.com` (custom domain) |

### Per-vertical pages (5)
Dynamic route `src/pages/pos-for-[vertical].astro` emits:
- `/pos-for-clothing`
- `/pos-for-retail`
- `/pos-for-restaurant`
- `/pos-for-hospital`
- `/pos-for-pharmacy`

Each has a unique `<title>`, meta description, canonical, vertical-tailored features, locked dashboard mock, vertical-specific FAQs, BreadcrumbList JSON-LD, and cross-links to the other 4 verticals. Content lives in `src/data/verticals.ts` as a typed single source of truth.

### Behavior & UX
- **Bilingual EN/SW toggle** — `[data-en]`/`[data-sw]` attributes on every text element; runtime swap supports inline HTML (`.sw` italic Swahili phrases stay rendered inside English paragraphs). Persists in `localStorage`.
- **Industry switcher** — clicking a tab rewrites category, mode, tag, products grid, net profit, sales today, transactions. Data map in `Layout.astro`.
- **FAQ accordion** — single-open with `max-height` transition.
- **Reveal-on-scroll** — IntersectionObserver + 900ms failsafe + `prefers-reduced-motion` override. SSR/SEO-safe.
- **Chat widget** — auto-opens 3s after first visit (once per session). Quick-reply chips on first greeting. Free-text + chip taps stream from the real Railway agent. WhatsApp fallback on any error.
- **Conversion events** — GA4 fires `whatsapp_click`, `app_cta_click`, `contact_email_click`, `chat_widget_open` via a single delegated click listener.

## Deviations from the original handoff

- **Stack:** Astro 4 with inline `<script>`s (not React islands w/ `client:visible`). Acceptable per the handoff — keeps the bundle tiny on a static marketing page.
- **Fonts:** Google Fonts `<link>` (not self-hosted `@fontsource-variable/plus-jakarta-sans`). Preconnects in place; self-hosting is a low-priority LCP optimization.
- **`<Image>` vs `<Picture>`:** Using `<Image>` (single `<img>` with WebP `srcset`), not `<Picture>` with AVIF+WebP type negotiation. WebP-only is fine in 2026 (~96% browser support).
- **Footer About / Privacy / Terms / Cookie:** Point to `/` placeholders per the client's choice for v1. Replace when real pages exist.
- **Testimonial avatar:** Fdhyler Collection brand logo at `public/testimonial-fdhyler.jpg`. The "F" initial remains as `onerror` fallback.
- **OG image:** Auto-generated from a temporary `/og-preview` route via headless Chrome. Has a tiny dev-toolbar sliver at the bottom — should be replaced with a designer-made asset.
- **Payments rebrand:** "Tigo Pesa" / "Tigo" replaced everywhere with **"Mixx by Yas"** (homepage chip, Features, Pricing, FAQ + JSON-LD, all 5 vertical pages, WhatsApp agent knowledge base).
- **Profit Spotlight chart:** Latent CSS bug fixed — bars were rendering at 0 px because their `height: %` had no resolved parent. Wrapped each `.bar` in a fixed-height `.bar-track`, and added per-day TZS values (Mon 138k → Sat 312k) below the day labels.
- **Tweaks panel:** Dropped (designer-only tool, not shipped).

## Build & deploy

```bash
npm run dev          # http://localhost:4321
npm run build        # → dist/  (also emits sitemap-index.xml + sitemap-0.xml)
npm run preview      # serve dist/ locally
```

CI: every push to `main` triggers `.github/workflows/deploy.yml` → builds → uploads `dist/` → deploys to GitHub Pages → live at `biashara-pos.com` within ~1–2 minutes. The Railway service for the WhatsApp agent auto-rebuilds from the same repo when `whatsapp-agent/` source changes.
